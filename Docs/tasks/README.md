# タスク管理

## 構成

- `current-sprint.md` - 現在進行中のスプリントタスク
- `be-task.md` - バックエンドの残タスク
- `fe-task.md` - フロントエンドタスク（作成予定）
- `completed-tasks.md` - 完了したタスクのアーカイブ

## 優先度の定義

- **高**: 1-2週間以内に完了すべきタスク
- **中**: 1ヶ月以内に完了すべきタスク
- **低**: 将来的に対応するタスク

## 進捗管理

毎週金曜日に以下を実施：

1. 完了したタスクを`completed-tasks.md`に移動
2. 次週のタスクを`current-sprint.md`に設定
3. 優先度の見直し
